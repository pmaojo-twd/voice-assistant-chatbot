# Iconografía

## Decisión: una sola librería — **Lucide**

El UI Kit en Figma incluye dos sets de iconos en paralelo (Lucide y Heroicons),
pero **el código del proyecto usa exclusivamente `lucide-react`** y los
componentes shadcn-style del repo importan iconos así:

```ts
import CheckIcon from "lucide-react/dist/esm/icons/check";
```

Razones:

1. **Consistencia con el código existente** (`button`, `select`, `dialog`, `carousel`, `sonner`, `checkbox` ya importan de Lucide).
2. **Tree-shaking explícito** vía import directo `lucide-react/dist/esm/icons/*`.
3. **Cobertura**: Lucide cubre todos los casos del UI Kit (navegación, comunicación, calendario, viaje, etc.).
4. **Configuración shadcn**: `components.json` tiene `"iconLibrary": "lucide"`.

> Si abres una PR con un icono de Heroicons, **conviértelo a su equivalente Lucide** antes de mergear.

## Tamaños

| Contexto | Clase | Tamaño |
|---|---|---|
| Inline en texto pequeño | `size-3` | 12px |
| Inline en texto base | `size-4` | 16px (default en `Button`) |
| Botón icon-sm | `size-4` | 16px |
| Botón icon-lg / avatares | `size-5` | 20px |
| Iconos hero / empty state | `size-12` | 48px |

## Reglas

- Usa `text-muted-foreground` para iconos decorativos.
- Usa el color del texto (`currentColor`) para iconos que comunican estado.
- En botones, no añadas clase de tamaño: el `Button` ya define `size-4` por default.
- Acompaña siempre los iconos sin label con `aria-label` o un `<span className="sr-only">`.

## Cómo añadir un icono nuevo

1. Búscalo en [lucide.dev/icons](https://lucide.dev/icons).
2. Importa con la ruta directa: `import FooIcon from "lucide-react/dist/esm/icons/foo";`.
3. Si el icono no existe en Lucide, abre issue antes de añadir Heroicons como excepción.
