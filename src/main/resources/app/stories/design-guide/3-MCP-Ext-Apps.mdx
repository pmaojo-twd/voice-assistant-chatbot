# Principios de diseno aplicados a MCP Apps

Estas reglas aterrizan como disenar UI dentro de una conversacion.

## 1) Extraer, no portar

**Aplicado:** una tarjeta para confirmar una accion o elegir entre opciones concretas.  
**No aplicado:** meter un dashboard entero con navegacion profunda dentro del chat.

## 2) Conversacion primero

**Aplicado:** respuestas cortas, acciones declarativas y siguientes pasos sugeridos.  
**No aplicado:** textos largos sin CTA ni direccion clara.

## 3) Jerarquia de modos

- `Inline`: micro tareas, feedback rapido, confirmaciones.
- `Carousel`: comparacion de alternativas.
- `Fullscreen`: tareas ricas o de multiples pasos.

## 4) Estado visible en todo momento

El usuario debe saber siempre si la herramienta esta:

- lista para actuar
- ejecutando una accion
- completada
- bloqueada por error recuperable

## 5) Accesibilidad operativa

- Foco de teclado visible (`ring`).
- Contraste consistente en claro/oscuro via tokens.
- Etiquetas y texto de accion explicitos.

## Mapeo rapido: principio -> componente

- Extraer, no portar -> `InlineCard`
- Comparar opciones -> `InlineCarousel`
- Flujo complejo -> `FullscreenView`
- Feedback de estado -> `Alert`, `Badge`
